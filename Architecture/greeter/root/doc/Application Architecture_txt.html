    Application Architecture¶ ↑

    We explain here how to write the documentation for your final project. As an example, we provide a simple Greeter web application which is just a glorified version of the classical Hello World program but running on the web and producing results in different natural languages.

    The directory structure for the application and its documentation is as follows:

    greeter/
      ├─ doc/                    (produced by RDoc)
      ├─ public/                 (folder for the server's public documents)
              └─ stylesheets/    (CSS files)
      ├─ models/                 (the application's model files)
      └─ views/                  (the application's view files)

    This is the command used to produce this documentation (running it from the greeter directory):

    rdoc --main "root/Application Architecture.txt" --exclude ".json|.css" root

    Patterns Used¶ ↑

    Briefly mention all the patterns that your application uses and identify where exactly. In our example, the following pattern are clearly used:

    • Simple Factory Pattern: The GreeterFactory is used to create Greeter instances by specifying the desired language during its creation.

    • Model-View-Controller: The application follows the classical web implementation of the MVC architectural pattern. The models (.rb files) and views (.erb files) are stored in the corresponding models and views directory. The controller is contained in server.rb file.
